# Codex 功能说明

更新时间：2026-07-20

这是一份通俗版说明，用来介绍 Codex 能做什么、适合怎么用，以及使用时需要注意哪些边界。你可以把 Codex 理解成一个能对话、能查资料、能处理文件、能操作本地项目、能辅助自动化工作的 AI 协作伙伴。

实际可用能力会受运行环境、账号权限、插件是否启用、网络状态、浏览器登录状态、MCP/连接器配置，以及文件访问权限影响。

## 1. 互动问答和思路整理

Codex 可以像普通 AI 助手一样进行互动问答，但重点不只是“回答一句话”，也能陪你把问题拆清楚、把想法变成可执行方案。

可以做：

- 解释概念：技术、工具、产品、流程、论文、政策等。
- 拆解问题：把模糊目标拆成步骤、优先级和行动清单。
- 辅助决策：整理选项、利弊、风险、成本和适用场景。
- 角色化沟通：帮你准备汇报、面试、客户沟通、邮件回复。
- 学习陪练：制定学习路线、生成练习题、解释错误原因。

适合这样提问：

- “用新手能懂的话解释一下这个概念。”
- “帮我把这个想法拆成一个可执行计划。”
- “我有三个方案，帮我比较一下哪个更稳。”

## 2. 本地项目处理

Codex 可以读取和操作本地工作区里的项目文件，适合处理代码、配置、文档、脚本、测试和项目结构。

可以做：

- 阅读项目结构，理解模块之间的关系。
- 修改代码、补功能、修 bug、重构小范围逻辑。
- 添加或更新测试，并运行测试命令验证结果。
- 检查配置文件、依赖文件、构建脚本和环境变量说明。
- 写 README、开发文档、变更说明、部署说明。
- 做代码评审，指出潜在 bug、风险、缺少测试的地方。
- 启动本地服务，打开页面检查功能和界面。

适合的任务例子：

- “帮我看一下这个项目怎么启动。”
- “修复这个报错，并跑测试确认。”
- “给这个项目补一个 README，让别人能看懂怎么用。”
- “review 一下这次改动，重点看风险。”

使用边界：

- Codex 不会主动撤销你已有的改动，除非你明确要求。
- 删除、重置、覆盖、批量移动等高风险操作需要非常明确的指令。
- 如果项目需要密钥、账号或外部服务，通常需要你提供合法访问方式或配置说明。

## 3. SSH 和远程环境协作

在允许使用 SSH 的环境里，Codex 可以帮助你连接远程服务器、排查服务问题、整理部署流程，或者把本地项目和远程环境串起来。

可以做：

- 查看远程目录、日志、配置、进程和磁盘空间。
- 协助部署项目、拉取代码、安装依赖、运行服务。
- 排查常见问题，比如端口占用、权限错误、服务启动失败。
- 编写部署脚本、维护脚本、备份脚本和操作说明。
- 对比本地和远程环境差异，找出配置不一致的地方。

适合的任务例子：

- “帮我看一下服务器上这个服务为什么起不来。”
- “把这个项目部署到远程机器，并写下步骤。”
- “检查一下日志里最近的错误，按原因分类。”

使用边界：

- SSH 需要已有账号、密钥、权限和网络连通性。
- 涉及生产环境、删除数据、重启关键服务、修改防火墙等操作，应先确认影响范围。
- Codex 可以帮你执行和检查，但不能替代运维审批、备份策略和安全规范。

## 4. 联网查资料和核验信息

Codex 可以搜索公开网页、官方文档、新闻、产品信息、技术资料、政策说明等，并把复杂内容整理成中文要点。

可以做：

- 查找并筛选可靠来源，优先使用官方文档、原始公告、论文、标准文档或权威来源。
- 对“最新”“今天”“最近”这类容易变化的信息进行实时核验。
- 把多篇资料整理成摘要、对比表、时间线、FAQ 或决策建议。
- 翻译、改写、提炼重点，生成适合阅读或汇报的版本。
- 对医疗、法律、金融等高风险话题保持谨慎，提示哪些地方需要专业人士确认。

适合的任务例子：

- “查一下这个工具最新版本有什么变化，整理成中文。”
- “比较这三个产品，按价格、功能、限制、适用人群做表格。”
- “帮我读这篇论文，讲人话，并列出值得关注的风险。”

## 5. 写作、整理和知识管理

Codex 可以把散乱信息整理成可保存、可复用的材料。

可以生成或整理：

- Markdown 文档、说明书、README、操作手册、会议纪要、复盘报告。
- 项目计划、需求文档、技术方案、产品说明、调研报告。
- 邮件、公告、简历、求职信、商务材料、培训材料。
- 读书笔记、课程笔记、知识库条目、问答库。
- 多种语气版本，比如正式版、口语版、老板能看懂版、客户能看懂版。

也可以做信息清洗：去重、归类、补标题、做目录、提炼结论、标出待确认问题。

## 6. Office、PDF 和表格处理

在具备相关工具的环境里，Codex 可以处理常见办公文件，不只是复制粘贴文字。

可以处理：

- Word / DOCX：创建文档、调整结构、润色、加批注、做修订说明、检查排版。
- PDF：提取文字和表格、总结内容、拆分合并、生成新 PDF、渲染页面检查效果。
- Excel / CSV / TSV：清洗数据、合并表、写公式、生成统计表、制作图表、检查异常值。
- PowerPoint：设计演示结构、生成页面内容、统一版式、制作汇报型或教学型幻灯片。

适合的任务例子：

- “把这几个 PDF 提炼成一个中文对比表。”
- “把这个 CSV 清洗一下，做成可以给业务看的 Excel。”
- “根据这份材料做一版 10 页以内的汇报 PPT。”

## 7. 数据分析、图表和交互可视化

Codex 可以把数据、日志、表格或复杂流程变得更直观。

可以做：

- 清洗数据：处理空值、重复值、格式混乱、编码问题、异常记录。
- 分析数据：分组统计、趋势分析、漏斗分析、相关性探索、简单预测。
- 生成图表：折线图、柱状图、散点图、热力图、时间线、流程图、地图等。
- 制作交互工具：仪表盘、模拟器、小型实验台、参数调节器、可视化解释页面。
- 解释结果：说明图里真正值得注意的变化、原因和不确定性。

适合的任务例子：

- “分析这个销售表，找出增长和下滑的原因。”
- “把这个算法过程做成一个能调参数的可视化页面。”
- “把日志里的错误按类型和时间统计出来。”

## 8. 浏览器操作、网页测试和界面检查

在具备浏览器控制能力的环境里，Codex 可以打开网页，像用户一样点击、输入、截图和检查界面。

可以做：

- 打开本地或远程网页，验证页面是否真的能用。
- 检查按钮、表单、导航、弹窗、登录后的页面状态。
- 做视觉检查：文字是否重叠、移动端是否错位、图片是否加载、画布是否空白。
- 用截图记录问题，帮助定位 UI bug。
- 对网页应用做基础回归测试，比如点击按钮后是否出现正确结果。

需要注意：如果网页需要登录，通常需要当前浏览器已有授权，或者提供合法访问方式。Codex 不会绕过登录、验证码或权限限制。

## 9. 图片、视觉素材和设计辅助

Codex 可以查看本地图片，也可以在适合的时候生成或编辑位图素材。

可以做：

- 生成封面图、插图、纹理、游戏素材、产品概念图、透明背景素材。
- 根据参考图做风格变体、颜色方案、构图建议。
- 检查图片里的文字、布局、清晰度、裁剪和视觉问题。
- 为网页、应用、PPT 或报告准备配图思路。

如果项目已有设计系统、图标库或 SVG 资源，通常应优先沿用已有风格，避免生成不一致的素材。

## 10. 定时提醒、自动化和持续跟进

在支持自动化的 Codex 环境里，可以设置提醒、周期检查或稍后继续工作。

常见用法：

- “明天上午提醒我继续改这份方案。”
- “每天早上检查某个网页有没有更新。”
- “每周五生成一次项目状态摘要。”
- “一个小时后继续这个任务，检查测试结果。”

自动化可以是独立运行的周期任务，也可以是当前任务里的后续提醒。是否可用取决于当前环境和权限。

## 11. 连接外部工具和私有数据

如果配置了 MCP 服务器、插件或应用连接器，Codex 可以接入更多工具和数据源，例如代码托管、文档系统、日历、消息系统、知识库、数据库、浏览器、设计工具等。

可以做：

- 查询私有仓库、工单、文档或会议记录。
- 从外部系统拉取数据，整理成报告。
- 把任务拆给不同工具执行，再合并结果。
- 根据团队已有流程减少手工复制粘贴。

前提是相关连接器已经安装、启用，并且当前账号有访问权限。

## 12. 管理和定制 Codex 的工作方式

Codex 也可以帮助配置自己的协作方式，让重复任务更稳定。

可以做：

- 编写或改进 `AGENTS.md`，记录项目约定、测试命令、代码风格和注意事项。
- 创建或安装技能，让 Codex 学会一套可复用的工作流程。
- 创建插件，把技能、工具、MCP、脚本和配置打包成可复用能力。
- 配置 hooks，在特定操作前后做检查或提醒。
- 管理 Codex 任务：创建新任务、跟踪后台任务、分叉任务、重命名、置顶、归档、交接任务。

如果经常重复交代同一套规则，可以把它沉淀成配置、技能或插件。

## 13. 质量检查、评审和风险提醒

Codex 可以作为“第二双眼睛”，帮助检查内容是否可靠、完整、可执行。

可以检查：

- 文档有没有逻辑断层、概念混用、前后矛盾。
- 表格数据有没有异常值、单位错误、公式问题。
- 网页或应用有没有明显的可用性、可访问性、移动端适配问题。
- 方案有没有风险、遗漏、依赖假设和验证路径。
- 命令或脚本是否可能误删文件、泄露密钥或影响生产环境。

适合的任务例子：

- “帮我 review 一下这个方案，重点看风险和遗漏。”
- “检查这个表格里的公式和异常数据。”
- “看一下这个脚本有没有危险操作。”

## 14. 更适合的提问方式

给 Codex 的任务越接近“目标 + 材料 + 约束 + 交付格式”，效果通常越好。

示例：

- “把这个文件夹里的 PDF 总结成一个 Markdown 表格，重点看价格和风险。”
- “做一个交互式网页，能调参数，看结果变化，移动端也要能用。”
- “帮我写一份 2 页以内的中文说明，语气专业但别太硬。”
- “检查这个 Excel，找出公式错误和异常数据，最后给我一个修复版。”
- “部署前帮我检查一下这套命令有没有风险。”

## 15. 重要边界

Codex 很适合协作处理复杂工作，但仍然需要明确边界。

- 需要权限：没有文件、账号、连接器或网络权限，就不能访问对应内容。
- 需要核验：价格、政策、新闻、接口文档、软件版本等可能变化的信息，应以实时资料为准。
- 需要保护工作区：删除、覆盖、重置、批量移动等危险操作，应先确认。
- 不能替代专业人士：医疗、法律、投资、税务等高风险决策，Codex 可以辅助理解信息和列问题，但不能替代专业意见。
- 结果最好验证：能跑测试、渲染截图、打开文件检查时，应尽量用实际结果确认。

## 最短版

Codex 不只是聊天助手。它可以互动问答、整理思路、处理本地项目、通过 SSH 协作远程环境、联网查资料、写文档、处理 Office/PDF/表格、分析数据、做图表和交互页面、测试网页、生成或检查图片、设置提醒和自动化、连接外部工具、管理任务、创建技能/插件，并帮助做质量检查和风险提醒。

更简单地说，Codex 是一个能理解目标、能动手执行、能检查结果、也能持续跟进的 AI 工作伙伴。
